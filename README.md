## Финальный рабочий процесс (рекомендуемый)
1. Картинки кладем локально в `company-site/assets/`.
2. Вносим правки в HTML/CSS/JS.
3. Проверяем в браузере через hard refresh (`Cmd + Shift + R`).
4. Делаем commit и push в GitHub.

## Важные правила для дальнейшей работы
- Все ассеты хранить в `assets/`, не разбрасывать по другим папкам.
- Перед публикацией проверять мобильный вид.
- Не делать слепую копию чужих лендингов: берем структуру и логику, но сохраняем уникальную реализацию.
- Каждую крупную правку фиксировать отдельным коммитом с понятным названием.

---
Если нужно, следующий шаг: сделать отдельный `CONTRIBUTING.md` с правилами именования файлов, веток и формата коммитов.

## Новое обязательное правило работы (Single Source of Truth)
- Начиная с этого этапа, любые изменения сайта делаем только в репозитории GitHub:
  - https://github.com/egorande228/website-mob-cash-lama
- Локальные папки вне репозитория не считаются актуальными и не используются как источник данных.
- Перед началом любой правки проверяем актуальное состояние `main` и работаем только с ним.
- После каждой логической правки: `commit` и `push`, чтобы в GitHub всегда была последняя версия.
